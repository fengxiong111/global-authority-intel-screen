# Authority Intel Screen 展示规范 v1

## 默认模式

默认只做一种展示模式：

- **Flow Mode**

不做第二套默认 UI。  
Edition Mode 只作为未来可能的增强方向，不进入 v1 主开发范围。

## 产品目标

这是一个持续更新的多源情报流终端，不是资讯门户，不是纸媒复刻，不是卡片社区。

默认目标只有四个：

1. 更快扫读
2. 更安静
3. 更少噪音
4. 更长期可读

## 默认阅读骨架

Flow Mode 必须满足：

- 黑底
- 左对齐
- 单列
- 大标题
- 极小时间
- 极轻来源
- 极轻分类
- 原生滚动
- 不显示摘要
- 不显示图片
- 不显示点赞 / 评论 / 热度数字
- 不做复杂动画
- 不做花哨装饰
- 不做资讯网站风格
- 不做卡片瀑布流
- 不做多栏报纸排版

## 信息层级

每条内容默认只展示：

1. 中文主标题
2. 时间
3. 来源
4. 分类（可选）

强弱关系必须固定：

- 标题是绝对主视觉
- 时间和来源显著弱于标题
- 分类只做辅助辨认，不能抢戏

## 内容规范

- 展示层目标是全中文主标题
- 数据层保留原始标题
- 点击跳转原文
- 默认不显示长摘要
- 默认不显示封面图
- 默认不显示作者头像
- 默认不显示评论区信息

允许的轻蒸馏：

- 标题去水分
- 中文统一
- 重复事件合并

不允许：

- 长摘要
- 观点输出
- 情绪化改写
- 偏离原意的重写

## 多端规则

### iPhone

- 永远单列
- 更大字号
- 更大行距
- 更少辅助信息

### iPad

- 默认仍为单列 Flow Mode
- 允许更宽内容区
- 允许头条更大
- 不做传统报纸分栏

### 27 寸显示器

- 默认仍为单列 Flow Mode
- 保持远距离可读性
- 可有“头条 + 连续流”终端感
- 不做报纸豆腐块

## 交互原则

- 默认原生滚动
- 不锁 body / html
- 不自定义触摸滚动
- 不做自动滚动特效
- 顶部分类切换必须极轻量
- 分类切换不能改变页面气质

## 验收标准

必须满足：

- 一眼先看到主标题
- 连续读 20 分钟不累
- 远看仍清楚
- iPhone / iPad / 27 寸都自然
- 没有门户站噪音
- 没有报纸排版断裂感
- 整体气质接近安静阅读型情报屏

## v1 结论

**Authority Intel Screen v1 默认展示规范 = Flow Mode。**

先把这一个模式做到极致，不并行开发第二套默认 UI。
